# 🔐 قواعد الحماية — Firebase Security Rules

## الخطوات (دقيقتان فقط)

1. افتح [Firebase Console](https://console.firebase.google.com)
2. اختر مشروعك: `university-master-portal`
3. من القائمة الجانبية: **Realtime Database** ← **Rules**
4. احذف القواعد الحالية والصق هذا بدلها:

```json
{
  "rules": {
    "applications": {
      "$dept": {
        ".read": true,
        ".write": "newData.child('secret').val() === 'UNV-MASTER-2026-SEC'"
      }
    }
  }
}
```

5. اضغط **Publish**

---

## ماذا تفعل هذه القواعد؟

| العملية | قبل الإصلاح | بعد الإصلاح |
|---------|-------------|-------------|
| **القراءة** | مفتوحة للجميع | مفتوحة للجميع (الطلاب يحتاجون رؤية أقسامهم) |
| **الكتابة** | مفتوحة للجميع | مقيّدة — يجب أن يحمل الطلب السر `UNV-MASTER-2026-SEC` |
| **التعديل** | مفتوح للجميع | مقيّد بنفس السر |
| **الحذف** | مفتوح للجميع | مقيّد — فقط عبر التطبيق الذي يعرف السر |

---

## ملاحظة أمنية

هذا الحل يوفر حماية **كافية** لمنع المتطفلين العشوائيين من الكتابة/الحذف.
للحماية المتقدمة (حيث حتى قراءة البيانات تتطلب مصادقة)، الخطوة التالية هي
إضافة Firebase Authentication للأدمن واستبدال السر بـ `auth != null`.

# 🔥 قواعد Firebase الصحيحة — افعل هذا الآن

## الخطوات (دقيقة واحدة)

1. افتح [Firebase Console](https://console.firebase.google.com)
2. اختر مشروعك: **university-master-portal**
3. من القائمة الجانبية: **Realtime Database** ← **Rules**
4. احذف كل شيء والصق هذا:

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

5. اضغط **Publish**

---

## لماذا تغيّرنا الـ Rules؟

الـ Rules السابقة (بالـ `secret` في الـ body) **لا تعمل** مع Firebase Realtime Database.
Firebase RTDB يتحقق من Rules **قبل** قراءة محتوى الطلب، فكان يرفض كل كتابة.

هذا هو سبب رسالة "حدث خطأ غير متوقع".

---

## ⚠️ ملاحظة أمنية

هذه القواعد المفتوحة مقبولة **مؤقتاً** لأن:
- البيانات ليست حساسة جداً (لا كلمات مرور، لا بطاقات بنكية)
- المنصة للاستخدام الجامعي المحدود
- الـ URL غير معروف للعموم

للحماية الكاملة لاحقاً: أضف **Firebase Authentication** للأدمن.
